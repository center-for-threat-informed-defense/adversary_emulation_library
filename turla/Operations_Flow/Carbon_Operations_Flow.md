# Carbon Emulation Scenario üìñ 
This scenario follows Turla‚Äôs multi-phase approach to implant a watering hole as a way to compromise more victims of interest. Turla gains initial access through a spearphishing email, a fake software installer is downloaded onto the victim machine and execution of the EPIC payload takes place. Once persistence and C2 communications are established, a domain controller is discovered, and CARBON-DLL is ingress into victim network. Further lateral movement takes the attackers to a Linux Apache server, PENGUIN is copied to the server and used to install a watering hole.

<br>**Phase 1:** The scenario begins when the user Gunter visiting a typosquat domain via a spearphishing email. When the link is opened, a fake software installer is downloaded on Gunter‚Äôs Windows host machine. Gunter clicks ‚ÄúInstall,‚Äù executing the malicious EPIC payload. EPIC establishes persistence and connects to a C2 server via a proxy web server using HTTP requests. The attacker‚Äôs enumeration on Gunter‚Äôs device, discovers a domain controller on the network and the domain admin Frieda.

<br>**Phase 2:** To maintain persistence, the attackers will deploy CARBON-DLL and prepare to laterally move onto the domain controller. The use of password spraying helps to obtain Frieda‚Äôs domain admin credentials. Frieda‚Äôs credentials are then used to mount the DC‚Äôs hard drive, move a copy of CARBON-DLL to this drive, and execute the malware on the DC via remote scheduled task. A Linux Apache Server and the server‚Äôs admin Adalwolfa is discovered during enumeration. CARBON-DLL downloads Mimikatz and dumps Adalwolfa‚Äôs stored NTLM hash. Mimikatz and Adalwolfa‚Äôs NTLM hash are used to perform a pass-the-hash (PtH) attack allowing the attackers to authenticate into Adalwolfa‚Äôs device and copy CARBON-DLL to the workstation via PsExec. 

<br>**Phase 3:** On Adalwolfa‚Äôs workstation, a custom keylogger is dropped and captures plaintext SSH credentials. CARBON-DLL downloads the PENQUIN malware to Adalwolfa‚Äôs workstation, the SSH credentials allows the attackers to copy PENQUIN to the Apache server and execute the malware via plink. PENQUIN is utilized to install a watering hole causing users browsing a legitimate HTML site to be redirected to the malicious, attacker-controlled site.
